# 大猫投资 — 游戏策划案 (Game Design Document)

> 基于代码逆向整理，标注了当前实现状态、设计漏洞、以及重构建议。

---

## 一、游戏概览

| 项目 | 说明 |
|------|------|
| 类型 | 多人实时股票模拟交易 + 公司经营 |
| 定位 | 教育性投资模拟游戏，侧重 A 股市场规则 |
| 玩家规模 | WebSocket 房间模式，单房间支持 50+ 人 |
| 市场周期 | 1 tick = 1.5 秒，1 季度 = 200 ticks = 5 分钟 |
| 核心循环 | 实时交易 → 资产增值 → 排行榜竞争 |

---

## 二、核心玩法

### 2.1 股票交易

玩家通过两个维度增值：

| 维度 | 操作 | 来源 |
|------|------|------|
| 价差交易 | 低买高卖 | 市场价格波动 |
| 股息收入 | 持有公司股票 | 公司季度利润分红 |

**交易类型：**

| 类型 | 描述 | 实现状态 |
|------|------|----------|
| 市价单 | 按当前市价即时成交 | ✅ 完整 |
| 限价单 | 指定价格挂单，等待匹配 | ✅ 完整 |
| 融资买入 (Margin) | 借钱加杠杆买 | ⚠️ 后端有，前端 UI 已移除 |
| 做空 (Short Sell) | 借股票卖出，低价买回 | ⚠️ 后端有，前端 UI 已移除 |

**费率体系（模拟 A 股）：**

| 费用 | 买入 | 卖出 | 费率 |
|------|------|------|------|
| 印花税 | 无 | 有 | 成交额 × 0.1% |
| 佣金 | 有 | 有 | 成交额 × 0.025%，最低 ¥5 |
| 融资利息 | - | - | 负债 × 0.0003%/20tick |
| 融券费用 | - | - | 持仓市值 × 0.0003%/20tick |

**交易限制（人类玩家）：**

| 限制 | 值 |
|------|-----|
| 单笔最大 | 流通股 × 1% = 500 万股 |
| 单人持仓上限 | 流通股 × 5% = 2500 万股 |
| 融券资格 | 总资产 ≥ ¥100 万 |
| 强制平仓线 | 担保比例 < 130% |
| 购买力 | (可用现金 - 冻结) × 2 |

### 2.2 公司经营

玩家可注册公司，享受完整的 P&L 模拟体验：

```
注册 → 选择行业 → 分配股权 → 季度经营 → 股价变动 → 分红 → 影响排名
```

**6 大行业:**

| 行业 | 特点 | 基础 PE |
|------|------|---------|
| 科技 (💻) | 高增长、高波动 | 25 |
| 金融 (🏦) | 稳定增长、低波动 | 12 |
| 制造 (🏭) | 稳定收益、周期性强 | 15 |
| 能源 (⚡) | 强周期、政策敏感 | 10 |
| 消费 (🛒) | 防御性、稳定现金流 | 20 |
| 医疗 (💊) | 防御性、高利润 | 30 |

**公司股票的估值公式：**
```
股价 = NAV + EPS × PE
NAV = 总资产 / 总股本
EPS = 净利润 / 总股本
PE = 行业基础PE × 周期系数
```

**经营决策维度：**

| 维度 | 方式 | 频率 |
|------|------|------|
| 策略选择 | 稳健 / 激进 / 成本优化 / 市场渗透 | 可随时调整 |
| 分配比例 | 储备 / 销售 / 分红 / 研发 (四项 0-100%) | 每季度 |
| 现金操作 | 回购、特别分红、招人、裁员、注资、营销、公关、并购、行业转型 (9 种) | 随时 |

**季度营收公式：**
```
营收 = 员工数 × 人均营收基数 × 周期系数 × 研发效率 × 规模系数
     × 行业趋势 × 销售加成 × 营销加成 × 随机(0.85-1.15) × 策略乘数
```

**破产机制：** 连续两个季度现金 < 0 → 重置为行业初始值。

### 2.3 AI 市场参与者

市场由 8 类 AI 玩家共同驱动，合计约 **210 个活动实体**：

| 类型 | 数量 | 角色 | 对市场影响 |
|------|------|------|-----------|
| 做市商多头 | 1 | 提供买盘流动性 | 价格地板 (5 档限价单) |
| 做市商空头 | 1 | 提供卖盘流动性 | 价格天花板 (持有 30% 流通股) |
| NPC 交易者 | 100 | 多样化策略 | 市场噪音 + 趋势 |
| 机构投资者 | 3 | 长线价值投资 | 稳定器 |
| 游资 | 5 | 快进快出投机 | 波动放大器 |
| 量化基金 | 5 | 算法交易 | 趋势+反转 |
| 国家队 | 4 | 托市 | 下跌保护 (-10%~-25% 触发) |
| 散户 | 100 | 情绪驱动 | FOMO + 恐慌 |

## 三、系统交互全景图

```
                    ┌──────────────┐
                    │   玩家交易   │
                    │  buy/sell    │
                    └──────┬───────┘
                           │ 订单流入
                    ┌──────▼───────┐
                    │  订单簿撮合   │
                    │  限价单匹配   │
                    └──────┬───────┘
                           │ 成交
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌─────────┐  ┌─────────┐  ┌─────────┐
        │ 价格更新 │  │ 成交量  │  │ Candle  │
        └────┬────┘  └────┬────┘  └────┬────┘
             │            │            │
             └────────────┼────────────┘
                          │
              ┌───────────┼───────────┐
              ▼           ▼           ▼
        ┌─────────┐ ┌─────────┐ ┌─────────┐
        │ AI Bot  │ │  SEC    │ │ 公司    │
        │ 决策    │ │ 监管    │ │ 季度    │
        └────┬────┘ └────┬────┘ └────┬────┘
             │           │           │
             └───────────┼───────────┘
                         │
              ┌──────────▼──────────┐
              │    WS 广播给所有玩家  │
              │   行情/持仓/榜单/新闻  │
              └─────────────────────┘
```

**关键设计原则：价格只通过实际交易变动，没有随机游走。** 这意味着市场动态完全由参与者的订单行为涌现。成交量不平衡漂移作为补充机制（买卖量差驱动小幅漂移，最大 ±0.05%/tick）。

## 四、游戏经济系统

### 4.1 货币流动

```
                          股票市场
                     买入 ↑    ↓ 卖出
                         ┌──────────┐
           个人现金  ←──→ │ 持仓市值  │
          (¥10,000起)     └──────────┘
               │               │
         注资  │               │ 股息
               ▼               ▼
            ┌──────────────────┐
            │    公司现金池     │
            │  (收入-成本+利息)  │
            └────┬───┬───┬─────┘
                 │   │   │
         回购/分红 招人/营销 科技投资
```

### 4.2 财富效应

| 机制 | 频率 | 影响 |
|------|------|------|
| 财富管理费 | 每季度 | 现金 > ¥10 万部分收 2% |
| 公司管理费 | 每季度 | 总资产 × 1% |
| 公司利息收入 | 每季度 | 现金余额 × 0.5% |
| 操盘手特权 | 无视 | admin@stock.game 起手 ¥5 亿 |

## 五、市场监管 (AI SEC)

### 违规检测

| 违规类型 | 阈值 | 处罚 |
|----------|------|------|
| 市场操纵 | 单人持仓 > 25% 流通股 | 累进罚款 + 交易限制 |
| 高频交易 | 60 tick 内 > 30 笔 | 累进罚款 + 交易限制 |

### 处罚梯度

| 次数 | 罚款 | 禁止交易 |
|------|------|----------|
| 第 1 次 | ¥10 万 | 0 季度 |
| 第 2 次 | ¥30 万 | 1 季度 |
| 第 3 次 | ¥80 万 | 2 季度 |
| 第 4+ 次 | ¥200 万 | 4 季度 |

罚款上限为 `min(罚金, 现金 × 0.5)`，最低 ¥1,000。

## 六、行业周期系统

每 4 季度检查一次：

```
正常 ──(动量 > 2.0)──→ 繁荣 ──(自动)──→ 正常
正常 ──(动量 < -2.0)──→ 衰退 ──(自动)──→ 正常
```

- 动量 = 累计随机游走 (−0.3 ~ +0.3/季度)
- **繁荣** 和 **衰退** 都是自限性的，持续 4 季度后自动回归正常
- 周期对营收的影响：繁荣 1.5x，正常 1.0x，衰退 0.65x

---

## 七、庄家操纵系统

4 阶段自动循环：

```
吸筹 (建仓) ──(price > MA20×1.03 或 >250 tick)──→ 拉升
拉升 ──(price > 目标价 或 >80 tick)──→ 洗盘
洗盘 ──(price < MA20×0.97 或 >50 tick)──→ 出货
出货 ──(持仓 < 5万 或 >200 tick)──→ 吸筹
```

## 八、设计评估：保留 vs 舍弃 vs 改造

### ✅ 核心保留（设计清晰，价值明确）

| 系统 | 理由 |
|------|------|
| 市价/限价交易 | 核心玩法 |
| 订单簿撮合 | 真实市场模拟的基石 |
| 做市商双头 | 提供基础流动性，机制简洁有效 |
| 国家队 4 档托市 | 有中国特色，防范极端行情 |
| 公司经营系统 | 独特的第二玩法维度 |
| 行业周期 | 增强公司经营的策略深度 |
| K 线图 (Canvas) | 性能好，功能完善，无需重写 |
| SEC 监管 | 防止单一玩家操纵市场 |

### 🔧 需要改造（方向对，实现有缺陷）

| 系统 | 问题 | 改造方向 |
|------|------|----------|
| **NPC 策略** | 100 个 NPC 但策略区分度低，行为趋同 | 减至 30-50 个，引入更多差异化参数（风险偏好、持股偏好、反应速度） |
| **庄家周期** | 4 阶段过于机械，所有股票同等对待 | 加入随机性，对成交量不足的股票降低操纵力度 |
| **新闻系统** | 生成频率低 (5 分钟冷却，2% 概率)，玩家感知不到 | 提高频率至 1-2 分钟一条，或取消冷却用泊松分布 |
| **成交量不平衡漂移** | 乘数 0.0005 太小，肉眼难以察觉 | 提高到 0.002-0.005，使其成为可观察的市场力量 |
| **财富管理费** | ¥10 万起征太低，惩罚小额玩家 | 提高到 ¥100 万，或改为累进税率 |
| **破产重置** | 现金 < 0 即重置，无缓冲 | 增加 1 次贷款/注资机会 |

### ❌ 建议舍弃

| 系统 | 理由 |
|------|------|
| **融资融券 (前端已移除)** | 前端 UI 已删，后端逻辑还在跑但无玩家使用。Go 迁移时可直接砍掉 |
| **AI 假成交记录** (`_ai_market_tape_trade`) | 纯粹是 UI 欺骗，制造虚假活跃度。应移除或用真实交替代替 |
| **操盘手 crash mode** | 管理员一键砸盘功能，破坏公平性。改为仅开发调试用 |
| **成就系统** | localStorage 实现，无服务器同步，无 UI 面板查看。应重新设计或砍掉 |
| **死订单簿渲染** (`renderOrderBook`) | 设计了精美 UI 但从未接入 HTML |
| **死新闻渲染** (`renderNews`) | 目标 DOM 不存在 | 
| **死聊天功能** | `case 'chat': break` 是空实现 |
| **空载首页** (lobby-page) | 完全不用的 div |
| **死后端指标** | 委比、委差、买卖量分拆 → 已推送到前端但从未展示 |

### ⚡ 待补完功能

| 功能 | 现状 | 建议 |
|------|------|------|
| 周/月 K 线 | 代码完整，无 UI 按钮 | React 版加按钮 |
| 指标切换 (MACD/KDJ/RSI/BOLL) | 代码完整，无 UI | React 版加按钮 |
| 公司公告发布 | 有 API + 弹窗，无触发按钮 | React 版加按钮 |
| 多股票支持 | 整个后端基于 `_any_stock_sym()` 只返回第一支 | 重构为多股票架构 |
| 资产曲线 | `equityCurve` 字段定义但从未填充 | 后端已有 `asset_history` 数据 |
| 受限价单列表 | `loadMyOrders()` 不存在 | 实现对应的 API 调用 |

## 九、重构优先级矩阵

以「游戏体验提升 / 实现成本」排序：

```
高影响低投入 ──→ 先做
  ├── 补完指标切换按钮
  ├── 补完周/月 K 线按钮
  ├── 修复新闻推送到正确 DOM
  ├── 提供可见订单簿 (接上现有 renderOrderBook)
  ├── 资产曲线图
  └── 修复公司排行榜 double revStr bug

高影响中投入 ──→ 第二阶段
  ├── 多股票支持 (后端核心改造)
  ├── 融资融券 UI 重新加入或彻底砍掉
  ├── NPC 策略差异化重写
  └── 庄家行为随机化

低影响低投入 ──→ 顺手修
  ├── 清除死代码 (lobby, chat, 旧 orderbook DOM)
  ├── 财富管理费起征点调高
  ├── 剔除 AI 假成交记录
  └── 密码验证文案对齐

中影响高投入 ──→ 重新评估
  ├── 公司系统复杂度（9 种现金操作是否都必要？）
  ├── 成就系统重新设计
  └── 新手引导重新制作
```

## 十、多股票架构设计方向

当前系统全是单股票硬编码 ("DM")。多股票支持需要解决：

| 问题 | 当前状态 | 目标 |
|------|----------|------|
| 做市商 | 仅交易 DM | 每支股票一对做市商 |
| NPC/Bot | `_any_stock_sym()` 只返回第一个 | 按流动性和认知分配 |
| K 线 | 单股票数据渲染 | 切换股票时重新加载数据 |
| 排行榜 | 按总资产排名 | 不变（平台无关） |
| 公司股票 | 已有多支，价基独立 | 保持不变 |
| 订单簿 | 每 tick 全量广播 | 按股票分发 |
| SEC | 只检查 DM | 跨股票持仓合并检查 |

**建议方案：** 将 DM 视为"主板"，加上玩家公司股票本身就构成了多股票市场。初期不引入更多 NPC 股票。重点是让做市商、Bot 和监管能处理多股票。

## 十一、设计哲学总结

1. **交易驱动价格，不靠随机数。** 价格由实际成交 + 成交量不平衡漂移决定。这使市场行为可以理解和预测。
2. **AI 提供流动性而非操纵。** 做市商的存在是为了让人类玩家可以交易，不是与玩家竞争。
3. **公司经营是第二增长曲线。** 股价不仅受交易影响，也受公司基本面驱动。持有自己公司的玩家有双重激励。
4. **监管是软性的。** SEC 只惩罚极端行为（25% 持仓、HFT），不限制正常交易。
5. **季节周期创造节奏。** 5 分钟一个季度形成自然的"回合"感，玩家需要定期做经营决策。
6. **国家队是安全网而非作弊。** 只在大跌时入场，且卖出时也是限价单，不影响正常行情。
