# 大猫投资 — 游戏策划案 (Game Design Document)

> 基于代码逆向整理，标注了当前实现状态、设计漏洞、以及重构建议。

---

## 一、游戏概览

| 项目 | 说明 |
|------|------|
| 类型 | 多人实时股票模拟交易 + 公司经营 |
| 定位 | 教育性投资模拟游戏，侧重 A 股市场规则 |
| 玩家规模 | WebSocket 房间模式，单房间支持 50+ 人 |
| 市场周期 | 1 tick = 1.5 秒，1 季度 = 200 ticks = 5 分钟 |
| 核心循环 | 实时交易 → 资产增值 → 排行榜竞争 |

## 二、核心玩法

### 2.1 股票交易

玩家通过两个维度增值：

| 维度 | 操作 | 来源 |
|------|------|------|
| 价差交易 | 低买高卖 | 市场价格波动 |
| 股息收入 | 持有公司股票 | 公司季度利润分红 |

**交易类型：**

| 类型 | 描述 | 实现状态 |
|------|------|----------|
| 市价单 | 按当前最优价位即时成交，可能滑点 | P3 实现 |
| 限价单 | 指定价格挂单，进入订单簿等待匹配 | P3 实现 |

**交易规则：**

| 参数 | 值 |
|------|-----|
| 交易 tick | 2 秒 |
| tick/季 | 150 tick（5 分钟） |
| 撮合规则 | 价格优先 → 时间优先 |
| 流通股来源 | IPO 增发股 → 证券机构库存 → 逐步释放到市场 |

**证券机构机制：**
IPO 增发的流通股先进入系统证券机构账户（BrokerInventory），不会一次性涌入市场。机构定时扫描超时未成交的买单，按最优价售出，将流通股逐步释放到玩家手中。

**费率体系（模拟 A 股）：**

| 费用 | 买入 | 卖出 | 费率 |
|------|------|------|------|
| 印花税 | 无 | 有 | 成交额 × 0.1% |
| 佣金 | 有 | 有 | 成交额 × 0.025%，最低 ¥5 |

**交易限制（人类玩家）：**

| 限制 | 值 |
|------|-----|
| 单笔最大 | 流通股 × 1% |
| 单人持仓上限 | 流通股 × 5% |

### 2.2 公司经营

玩家可注册公司，享受完整的 P&L 模拟体验：

```
注册 → 选择行业 → 确定股权结构 → 季度经营 → 达成 IPO 条件 → 上市 → 股票交易 → 股价变动 → 影响排名
```

**6 大行业:**

| 行业 | 特点 | 基础 PE |
|------|------|---------|
| 科技 (💻) | 高增长、高波动 | 25 |
| 金融 (🏦) | 稳定增长、低波动 | 12 |
| 制造 (🏭) | 稳定收益、周期性强 | 12 |
| 矿业 (⛏️) | 强周期、资源消耗型、越挖越少 | 10 |
| 消费 (🛒) | 防御性、稳定现金流 | 20 |
| 医疗 (💊) | 防御性、高利润 | 30 |

**公司估值公式（IPO 发行价 / 理论股价）：**
```
总资产 = Cash + CapCount × CapAssetValue
NAV    = 总资产 / TotalShares
EPS    = 近4季度平均净利润 / TotalShares
股价   = max(1, NAV + EPS × 行业PE × 景气度)
```

> **IPO 条件**：运营 ≥ 12 季、连续盈利 ≥ 4 季、现金 ≥ ¥100 万、近4季营收合计 ≥ ¥500 万。详见 `COMPANY_V2_DESIGN.md` §十六。

**股权结构模型：**

```
TotalShares = CEOShares(100,000) + InvestorShares(投资方) + PublicFloat(IPO增发流通股)
```

IPO 后 CEO 持股转为个人股票持仓（可自由交易），投资方持股同步进入证券机构库存，与流通股一起逐步释放到二级市场。

**经营决策维度：**

| 维度 | 方式 | 频率 |
|------|------|------|
| 扩产 | 每行业扩产形式不同（制造=建产线, 矿业=探矿脉），进入建造队列，完成后 `CapCount` 增加 | 每季最多 3 次 action |
| 招募 | 发布 N 个岗位，实际招到 `N × (30%~100%)`，成本 = `N × HireCost` | 每季最多 3 次 action |
| 裁员 | 辞退 N 名员工，成本 = `N × LaborRate × 3`（3倍季度工资补偿），可裁至 0 人 | 每季最多 3 次 action |
| 资产处置 | 折价 75% 出售产能（制造=卖产线, 矿业=卖矿权），获得 `N × CapAssetValue × 0.75` 现金 | 每季最多 3 次 action |
| IPO | 达成条件后发起，选择增发比例 | 一次性 |

> **扩产概念统一**：探矿不是行业特殊操作，而是矿业的扩产形式。各行业扩产名称不同、机制不同，但都是 `type: "expand"` 的基础扩产行为。

**破产机制：** 连续两个季度现金 < 0 → 重置为行业初始值。

### 2.3 AI 市场参与者

> 详细设计见 `AI_TRADER_DESIGN.md`。

市场由 100 个 AI 交易者按统一因子评分模板驱动。不区分类别，所有 AI 是同一类型实例，差异仅在于 7 种策略权重向量的不同配置。

| 策略 | 数量 | 风格 | 对市场影响 |
|------|:---:|------|-----------|
| value (价值) | 20 | 寻找低估，持有等回归 | 基本面锚定 |
| growth (成长) | 17 | 追增长，轻估值 | 成长股溢价 |
| momentum (动量) | 14 | 趋势交易，追涨放量 | 趋势放大器 |
| contrarian (逆向) | 12 | 反向操作，跌买涨卖 | 均值回归力 |
| balanced (均衡) | 23 | 多因子均衡 | 市场噪音基础 |
| national (国家队) | 3 | 托市 + 价值锚 | 下跌保护 |
| noise (噪声) | 11 | 纯随机 | 随机扰动 |

**核心机制**：12 个因子（6 理性 + 5 行为偏差 + 1 噪声）× 策略权重 → 综合信号 → 限价下单。市场情绪指数（Information Cascade 模型）提供 15% 的跨 AI 情绪传导。AI 零持仓起步，资金耗尽后退出并由系统自动补充。

```
             │            │            │
             └────────────┼────────────┘

             │           │           │
             └───────────┼───────────┘
```

**关键设计原则：价格只通过实际交易变动，没有随机游走。** 这意味着市场动态完全由参与者的订单行为涌现。成交量不平衡漂移作为补充机制（买卖量差驱动小幅漂移，最大 ±0.05%/tick）。

## 四、游戏经济系统

### 4.1 货币流动

```
                          股票市场
                     买入 ↑    ↓ 卖出
                         ┌──────────┐
           个人现金  ←──→ │ 持仓市值  │
          (¥10,000起)     └──────────┘
               │               │
         注资  │               │ 股息
               ▼               ▼
```

| 机制 | 频率 | 影响 |

| 公司利息收入 | 每季度 | 现金余额 × 0.5% |

## 五、市场监管 (AI SEC)

### 违规检测

| 违规类型 | 阈值 | 处罚 |
|----------|------|------|
| 市场操纵 | 单人持仓 > 25% 流通股 | 累进罚款 + 交易限制 |
| 高频交易 | 60 tick 内 > 30 笔 | 累进罚款 + 交易限制 |

### 处罚梯度

| 次数 | 罚款 | 禁止交易 |
|------|------|----------|
| 第 1 次 | ¥10 万 | 0 季度 |
| 第 2 次 | ¥30 万 | 1 季度 |
| 第 3 次 | ¥80 万 | 2 季度 |
| 第 4+ 次 | ¥200 万 | 4 季度 |

罚款上限为 `min(罚金, 现金 × 0.5)`，最低 ¥1,000。

## 六、行业周期系统

每 4 季度检查一次：

```
正常 ──(动量 > 2.0)──→ 繁荣 ──(自动)──→ 正常
正常 ──(动量 < -2.0)──→ 衰退 ──(自动)──→ 正常
```

- 动量 = 累计随机游走 (−0.3 ~ +0.3/季度)
- **繁荣** 和 **衰退** 都是自限性的，持续 4 季度后自动回归正常
- 周期对营收的影响：繁荣 1.5x，正常 1.0x，衰退 0.65x

## 七、庄家操纵系统

4 阶段自动循环：

```
吸筹 (建仓) ──(price > MA20×1.03 或 >250 tick)──→ 拉升
拉升 ──(price > 目标价 或 >80 tick)──→ 洗盘
洗盘 ──(price < MA20×0.97 或 >50 tick)──→ 出货
出货 ──(持仓 < 5万 或 >200 tick)──→ 吸筹
```

## 八、设计评估：保留 vs 舍弃 vs 改造

### ✅ 核心保留（设计清晰，价值明确）

| 系统 | 理由 |
|------|------|
| 市价/限价交易 | 核心玩法 |
| 订单簿撮合 | 真实市场模拟的基石 |
| 做市商双头 | 提供基础流动性，机制简洁有效 |
| 国家队 4 档托市 | 有中国特色，防范极端行情 |
| 公司经营系统 | 独特的第二玩法维度 |
| 行业周期 | 增强公司经营的策略深度 |
| K 线图 (Canvas) | 性能好，功能完善，无需重写 |
| SEC 监管 | 防止单一玩家操纵市场 |

### 🔧 需要改造（方向对，实现有缺陷）

| 系统 | 问题 | 改造方向 |
|------|------|----------|
| **NPC 策略** | 100 个 NPC 但策略区分度低，行为趋同 | 减至 30-50 个，引入更多差异化参数（风险偏好、持股偏好、反应速度） |
| **庄家周期** | 4 阶段过于机械，所有股票同等对待 | 加入随机性，对成交量不足的股票降低操纵力度 |
| **新闻系统** | 生成频率低 (5 分钟冷却，2% 概率)，玩家感知不到 | 提高频率至 1-2 分钟一条，或取消冷却用泊松分布 |
| **成交量不平衡漂移** | 乘数 0.0005 太小，肉眼难以察觉 | 提高到 0.002-0.005，使其成为可观察的市场力量 |
| **财富管理费** | ¥10 万起征太低，惩罚小额玩家 | 提高到 ¥100 万，或改为累进税率 |
| **破产重置** | 现金 < 0 即重置，无缓冲 | 增加 1 次贷款/注资机会 |

### ❌ 建议舍弃

| 系统 | 理由 |
|------|------|
| **融资融券 (前端已移除)** | 前端 UI 已删，后端逻辑还在跑但无玩家使用。Go 迁移时可直接砍掉 |
| **AI 假成交记录** (`_ai_market_tape_trade`) | 纯粹是 UI 欺骗，制造虚假活跃度。应移除或用真实交替代替 |

| **死订单簿渲染** (`renderOrderBook`) | 设计了精美 UI 但从未接入 HTML |
| **死新闻渲染** (`renderNews`) | 目标 DOM 不存在 |

| **死后端指标** | 委比、委差、买卖量分拆 → 已推送到前端但从未展示 |

## 九、重构优先级矩阵

以「游戏体验提升 / 实现成本」排序：

```
高影响低投入 ──→ 先做
  ├── 补完指标切换按钮
  ├── 补完周/月 K 线按钮
  ├── 修复新闻推送到正确 DOM
  ├── 提供可见订单簿 (接上现有 renderOrderBook)
  ├── 资产曲线图
  └── 修复公司排行榜 double revStr bug

高影响中投入 ──→ 第二阶段
  ├── 多股票支持 (后端核心改造)
  ├── 融资融券 UI 重新加入或彻底砍掉
  ├── NPC 策略差异化重写
  └── 庄家行为随机化

低影响低投入 ──→ 顺手修
  ├── 清除死代码 (lobby, chat, 旧 orderbook DOM)
  ├── 财富管理费起征点调高
  ├── 剔除 AI 假成交记录
  └── 密码验证文案对齐

中影响高投入 ──→ 重新评估
  ├── 公司系统复杂度（9 种现金操作是否都必要？）
  ├── 成就系统重新设计
  └── 新手引导重新制作
```

## 十、多股票架构设计方向

新系统以玩家公司为股票来源，每支 IPO 上市的公司即为一支可交易的股票。DM 主板股票不再保留。

| 问题 | 目标 |
|------|------|
| 股票来源 | 玩家公司 IPO 增发 → 流通股入市 |
| 做市商 | 每支已上市股票一对做市商（买/卖） |
| 证券机构 | 每支股票一个机构账户，持有并逐步释放 IPO 流通股 |
| NPC/Bot | 按流动性和市值分配交易权重 |
| K 线 | 切换股票时重新加载对应数据 |
| 排行榜 | 按总资产排名（跨股票） |
| 订单簿 | 按股票独立维护，按股票分发 |
| SEC | 跨股票持仓合并检查（P5） |

## 十一、设计哲学总结

1. **交易驱动价格，不靠随机数。** 价格由实际成交 + 成交量不平衡漂移决定。这使市场行为可以理解和预测。
2. **AI 提供流动性而非操纵。** 做市商的存在是为了让人类玩家可以交易，不是与玩家竞争。
3. **公司经营是第二增长曲线。** 股价不仅受交易影响，也受公司基本面驱动。持有自己公司的玩家有双重激励。
4. **监管是软性的。** SEC 只惩罚极端行为（25% 持仓、HFT），不限制正常交易。
5. **季节周期创造节奏。** 5 分钟一个季度形成自然的"回合"感，玩家需要定期做经营决策。
6. **国家队是安全网而非作弊。** 只在大跌时入场，且卖出时也是限价单，不影响正常行情。
